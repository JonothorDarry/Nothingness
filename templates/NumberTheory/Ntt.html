Introduction

This article describes techniques of multiplying two polynominals of size \(o, m\) in \(O((o+m)\log(o+m))\) and use of those techniques for solving problems regarding generating functions.

Problem I - multiplying polynominals

Suppose You are given coefficients \(a_0, a_1, \dots, a_o\) and \(b_0, b_1, \dots, b_m\) of two polynominals \(A(x), B(x)\), \(1 \le o,m,a_i,b_i \le 10^5\), and Your aim is to multiply them; in other words, You have to find such sequence \(c_0, c_1, \dots, c_{o+m}\) such that \(A(x)B(x)=(a_0+a_1x+a_2x^2+\dots+a_ox^o)(b_0+b_1x+b_2x^2+\dots+b_mx^m)=(c_0+c_1x+c_2x^2+\dots+c_{m+o}x^{m+o})=C(x)\). This problem can be solved in at least \(2\) versions: when You have to calculate coefficients as they are, or \(\pmod x\), where \(x\) has a primitive root and \(\exists_k 2^k|\phi(x) \land 2^k \gt m+o\) and \(2 \nmid x\)

First solution

One can multiply each element in the first bracket with element from second bracket, then add resulting \(om\) elements to correct coefficient \(c_i\); this solution works in \(O(om)\)

Threshold of transformation - solution in \(O((o+m)\log(o+m))\)

  1. If \(m+o=k\), then let \(n\) be minimal number, such that \(n \gt k \land \exists_{l \in Z} n=2^l\). Then, the result will be calculated as \(n\) coefficients, and polynominals \(A(x), B(x)\) will be used as if they had \(n\) coefficients (The additional coefficients, say, for \(A(x)\) they will be \(a_{o+1}, a_{o+2} \dots a_{n-1}\) will be evaluated to \(0\))
  2. One may change the methodology of obtaining a polynominal a little bit: instead of directly finding the coefficients of \(C(x)\), one may find \(n\) values \(x_i\), obtain values of both \(A(x_i)\) and \(B(x_i)\) in those points, then multiply values \(A(x_i)B(x_i)=C(x_i)=y_i\), and transform \(n\) values of polynominal \(C(x)\) in points \(x_i: C(x_i)=y_i\) into its coefficients \(c_i\) - this process is called interpolation of a polynominal.
  3. Let \(A(x)=a_0+a_1x+ \dots + a_{n-1}x^{n-1}\)
    \(A_1(x)=a_1+a_3x+a_5x^2+ \dots a_{n-1}x^{n/2-1}\)
    \(A_0(x)=a_0+a_2x+a_4x^2+ \dots a_{n-2}x^{n/2-1}\)     What follows, \(A(x)=A_0(x^2)+xA_1(x^2)\) What follows, one can calculate recursively values of polynominals \(A_0\),\(A_1\), then find the value of \(A(x)\) - the process will take \(O(nlog(n))\), if one problem will be addressed -Â if \(A_0(x_i)\) was found, then the values of \(A(x)\) that can be easily calculated are \(A(x_i^{1/2})\) and \(A(-(x_i^{1/2}))\) - so one needs to find suitable points for evaluating polynominal for subsequent iterations.

  4. Let \(w_n^k=f^{\frac{k}{n}}\), where \(f\) is some predefined value. Then, if in \(i\)-th iteration one uses values \(w_{n/2}^0, w_{n/2}^1, \dots, w_{n/2}^{n/2}\), in next iteration one can use values \(w_n^0, w_n^1, \dots, w_n^{n/2}\) without any problems as next places, in which value of polynominal will be calculated. \(h \le \frac{n}{2} \implies A(w_n^h)=A_0(w_n^{2h})+w_n^hA_1(w_n^{2h})=A_0(w_{n/2}^h)+w_n^hA_1(w_{n/2}^h)\) Furthermore, \(w_n^{s+n/2}=w_n^{n/2}w_n^s\), where \(s \le \frac{n}{2}\); this leads to establish \(w_n^{n/2}=-1\), and \(f=w_n^n=1\); then, \(w_n^{s+n/2}=w_n^{n/2}w_n^s=-w_n^s\), and \(w_n^{2(s+n/2)}=w_n^nw_n^{2s}=1w_{n/2}^s\) - so, one can calculate easily values of polynominal \(A(x)\) in \(n\) points if values of \(A_0(x), A_1(x)\) in certain \(\frac{n}{2}\) points were calculated. \(h=\frac{n}{2}+s \gt \frac{n}{2} \implies A(w_n^h)=A_0(w_n^{2h})+w_n^hA_1(w_n^{2h})=A_0(w_n^{2(n/2+s)})+w_n^hA_1(w_n^{2(n/2+s)})=A_0(w_n^nw_n^{2s})+w_n^hA_1(w_n^nw_n^{2s})=A_0(w_n^{2s})+w_n^hA_1(w_n^{2s})=A_0(w_{n/2}^s)+w_n^hA_1(w_{n/2}^s)\) All roots \(w_n^k\) have to be different, otherwise the last part of the algorithm will not work (because many polynominals of degree \( \lt n\) can have same values in \(\lt n\) points and the result would be ambiguous).

  5. It may be beneficial to get rid of recursion before things start getting messy; to find value of \(A(x)\) in \(n\) points, one needs to find values of \(A_0(x)=a_0+a_2x+\dots+a_{n-2}x^{n/2-1}\) and \(A_1(x)=a_1+a_3x+\dots+a_{n-1}x^{n/2-1}\) in \(\frac{n}{2}\) points. Then, to find values of \(A_0(x)\) one needs to find values of \(a_0+a_4x+a_8x^2+\dots+a_{n-4}x^{n/4}\) and \(a_2+a_6x+a_{10}x^2+\dots+a_{n-2}x^{n/4}\) in \(\frac{n}{4}\) points, etc.

    One can generalize the notion of polynominal on each level of recursion: if \(n=2^k\), then let \(A_{k,0}(x)\) be the polynominal given at the start; then, one can define \(A_{lev,pol}\) as \(pol\)-th polynominal calculated on \(lev\)-th level of recursion. \(A_{lev,pol}(x)=A_{lev-1,2pol}(x^2)+xA_{lev-1,2pol+1}(x^2)\) - the \(2pol\) and \(2pol+1\) emerge from fact, that every previous polynominal on the same level has exactly 2 polynominals on lower level. There will be \(k\) levels of recursion, on \(lev\)-th each level polynominal consists of \(2^{lev}\) coefficients (perhaps with leading 0's) - on lowest level of recursion it will consist of one coefficient, so its value is constant, equal to this coefficient.

    For example, for \(n=16\), the subsequent coefficients of polynominals in recursion will be: {% set pows=[1, 2, 4, 8, 16] %} {% set ns = namespace(start=0, broke=1) %} {% for cur in range(0, 5) %} {% set x=pows[cur] %} \( {% set ns.start=0 %} {% for i in range(0, x) %} {% set y=(16//x) %} ( {% for j in range(0, y) %} a_{ {{ j*x+ns.start }} } {% if j!=y-1 %} , {% endif %} {% endfor %} {% set ns.broke=1 %} {% for j in range(cur-1, -1, -1) %} {% if ji_ge(ns.start, pows[j]) and ns.broke==1 %} {% set ns.start=ns.start-pows[j] %} {% elif ns.broke==1 %} {% set ns.start=ns.start+pows[j] %} {% set ns.broke=0 %} {% endif %} {% endfor %} ) {% endfor %} \) {% endfor %}
    But only on the lowest level of recursion those coefficients are useful; in higher levels of recursion finding values of a polynominal in a point - is only dependant on that point and previous values of polynominals: \(A_{lev,pol}(x)=A_{lev-1,2pol}(x^2)+xA_{lev-1,2pol+1}(x^2)\). On a lowest level, value of a polynominal in form \(A_{0,pol}(x)=a_i\) is the coefficient itself; the question arises, how to construct such sequence as in the lowest level of recursion? This allows to process subsequent polynominals as pair of two next polynominals from level below. Let group mean sequence of coefficients in a polynominal on some level. One can observe, that:
    • In a group there are subsequent numbers with same difference between them: At the start, there is one group with difference \(1\); in each partition, a group is divided into two groups with every second element each; as the difference is constant between each - say \(d\), then in group on lower level there will be subsequent numbers with difference \(2d\). Furhermore, difference within one group is equal to \(2^l\), as on the highest level difference is \(1\), and in each lower level the difference becomes twice higher.
    • As in each group \(g_0\) there are subsequent elements with difference \(d\), then after division into groups \(g_1\) and \(g_2\) difference between \(k\)-th element of \(g_2\) and \(g_1\) is equal to \(d=2^{l-1}\), as subsequent elements from \(g_0\) go to different groups. Furthermore, both groups since then are divided in same way - then this holds after subsequent division and going deeper into recursion (that \(g_2(k)-g_1(k)=2^{l-1}\)).
    So, one can create such algorithm for attaining subsequent numbers in this seqence, call it \(S\) with element \(S(i)\): at the beginning, \(S(0)=0\), then, whenever \(2^x\) for some \(x\) is reached, then all subsequent elements up to \(2^{x+1}-1\) are evaluated as \(S(i)=S(i-2^x)+2^{k-x-1}\), where \(2^{k}=n\); the \(2^{k-x-1}\) follows from fact, that \(k-x-1\) is number of splits into \(2\) groups before this one.

  6. After finding salvation from abyss of abstract, absolutely brutal and unforgiving realm of finding values of two polynominals in \(n\) points in \(O(nlog(n))\), it may be wise to multiply values of both polynominals in those points: \(C(x_i)=A(x_i)B(x_i)\) - this step works in \(O(n)\)
  7. The final problem is finding from values \(C(x_0), C(x_1), \dots C(x_n)\) coefficients \(c_0, c_1, \dots, c_{n-1}\) of a polynominal \(C(x)\). One can represent the problem as matrix equation: $$ \begin{bmatrix} w_n^0 & w_n^0 & w_n^0 & \cdots & w_n^0\\ w_n^0 & w_n^1 & w_n^2 & \cdots & w_n^{n-1}\\ w_n^0 & w_n^2 & w_n^4 & \cdots & w_n^{2(n-1)}\\ w_n^0 & w_n^3 & w_n^6 & \cdots & w_n^{3(n-1)}\\ \vdots & \vdots & \vdots & \ddots & \vdots\\ w_n^{0} & w_n^{n-1} & w_n^{2(n-1)} & \cdots & w_n^{(n-1)(n-1)}\\ \end{bmatrix} \begin{bmatrix} c_0 \\ c_1 \\ c_2 \\ c_3 \\ \vdots \\ c_{n-1} \\ \end{bmatrix} = \begin{bmatrix} y_0 \\ y_1 \\ y_2 \\ y_3 \\ \vdots \\ y_{n-1} \\ \end{bmatrix} $$ $$ \begin{bmatrix} c_0 \\ c_1 \\ c_2 \\ c_3 \\ \vdots \\ c_{n-1} \\ \end{bmatrix} = \begin{bmatrix} w_n^0 & w_n^0 & w_n^0 & \cdots & w_n^0\\ w_n^0 & w_n^1 & w_n^2 & \cdots & w_n^{n-1}\\ w_n^0 & w_n^2 & w_n^4 & \cdots & w_n^{2(n-1)}\\ w_n^0 & w_n^3 & w_n^6 & \cdots & w_n^{3(n-1)}\\ \vdots & \vdots & \vdots & \ddots & \vdots\\ w_n^{0} & w_n^{n-1} & w_n^{2(n-1)} & \cdots & w_n^{(n-1)(n-1)}\\ \end{bmatrix}^{-1} \begin{bmatrix} y_0 \\ y_1 \\ y_2 \\ y_3 \\ \vdots \\ y_{n-1} \\ \end{bmatrix} $$ So, to solve given problem, one can find certain inverse matrix - if one exists, then there is only one. One can show, that: $$ A^{-1}= \begin{bmatrix} w_n^0 & w_n^0 & w_n^0 & \cdots & w_n^0\\ w_n^0 & w_n^1 & w_n^2 & \cdots & w_n^{n-1}\\ w_n^0 & w_n^2 & w_n^4 & \cdots & w_n^{2(n-1)}\\ w_n^0 & w_n^3 & w_n^6 & \cdots & w_n^{3(n-1)}\\ \vdots & \vdots & \vdots & \ddots & \vdots\\ w_n^{0} & w_n^{n-1} & w_n^{2(n-1)} & \cdots & w_n^{(n-1)(n-1)}\\ \end{bmatrix}^{-1} = \frac{1}{n} \begin{bmatrix} w_n^0 & w_n^0 & w_n^0 & \cdots & w_n^0\\ w_n^0 & w_n^{-1} & w_n^{-2} & \cdots & w_n^{-(n-1)}\\ w_n^0 & w_n^{-2} & w_n^{-4} & \cdots & w_n^{-2(n-1)}\\ w_n^0 & w_n^{-3} & w_n^{-6} & \cdots & w_n^{-3(n-1)}\\ \vdots & \vdots & \vdots & \ddots & \vdots\\ w_n^{0} & w_n^{-(n-1)} & w_n^{-2(n-1)} & \cdots & w_n^{-(n-1)(n-1)}\\ \end{bmatrix} = B $$ This equality stems from several theses (where \(AB=M\)):
    • The value of \(M[i,j]\) will be equal to \(M[i,j]=\frac{1}{n}(A[i,0]B[0,j]+A[i,1]B[1,j]+A[i,2]B[2,j]+ \dots + A[i,n-1]B[n-1,j])=\frac{1}{n}(w_n^0w_n^0+w_n^iw_n^{-j}+w_n^{2i}w_n^{-2j}+ \dots +w_n^{(n-1)i}w_n^{-(n-1)j})=\)\(=\frac{1}{n}(w_n^{0(i-j)}+w_n^{1(i-j)}+w_n^{2(i-j)}+ \dots +w_n^{(n-1)(i-j)})=\frac{1}{n}\sum_{k=0}^{n-1}w_n^{k(i-j)}\)
    • Let \(c=i-j\); as \(\forall_{x \in Z} w_n^x=1 \iff x \equiv 0 \pmod n\) (this emerges from fact, that all roots have to be different - and this shows, why roots have to be different), then what can be said about \(s: w_n^{sc}=1\)? Clearly, \(sc=h*lcm(n, c)=h\frac{nc}{gcd(n, c)}\) for some natural \(h\). A minimal \(s \gt 0\) is \(s=\frac{n}{gcd(n, c)}\). Certainly, \(s|n\) - and so, \(s=2^f\) for some natural \(f\). Furthermore, \(s=1\) if and only if \(gcd(n, c)=n\).
    • What can be said about \(c\)? as \(-n \lt i-j=c \lt n\), then \(gcd(n, c)=n \iff i-j=c=0\). In this case, $$\frac{1}{n}\sum_{k=0}^{n-1}w_n^{k(i-j)}=\frac{1}{n}\sum_{k=0}^{n-1}w_n^0=\frac{1}{n}\sum_{k=0}^{n-1}1=\frac{1}{n}n=1$$ - this is the value of the main diagonal \(M[i,i]\) of resulting matrix.
    • If \(c \neq 0\), then \(w_n^{cs/2}=-1\) - because \(2|s\) and \(w_n^{cs/2} \neq 1\) (by definition of \(s\)). In this case: {% set beg="\\frac{1}{n}\sum_{k=0}^{n/s-1}"%} $$M[i,j]=\frac{1}{n}\sum_{k=0}^{n-1}w_n^{(i-j)k}={{beg}} \sum_{l=0}^{s-1}w_n^{c(ks+l)}={{beg}} \sum_{l=0}^{s-1}(w_n^{cs})^{k}w_n^{cl}={{beg}} \sum_{l=0}^{s-1}w_n^{cl}={{beg}} \sum_{l=0}^{s/2-1}(w_n^{cl}+w_n^{s/2+cl})={{beg}} \sum_{l=0}^{s/2-1}(w_n^{cl}-w_n^{cl})={{beg}} \sum_{l=0}^{s/2-1}0=0$$
    And so, M is indeed identity matrix, so \(B=A^{-1}\)
  8. From inverse matrix equation, one may notice, that to calculate \(c_i\) is to solve following equation: $$c_i=\frac{1}{n} \sum_{j=0}^{n-1}y_jw_n^{-ij}$$ First of all, \(w_n^{-i}w_n^{i}=1=w_n^n=w_n^iw_n^{n-i}\). If \(w_n^{i}\) has exactly one inverse, then \(w_n^{n-i}=w_n^{-i}\). Secondly, the values calculated in the first phase of the algorithm had form: $$P(w_n^i)=\sum_{j=0}^{n-1}a_jw_n^{ij}$$ The only differences are: lack of division by \(n\), and \(w_n^{ij}\) instead of \(w_n^{-ij}\) - different roots, but properties of \(w_n^0=w_n^n=w_n^{-n}=1\) and \(w_n^{n/2}=w_n^{n-n/2}=w_n^{-n/2}\) stay the same - this allows one to construct algorithm finding \(c_i\) in \(O(nlog(n))\) in the same way as for finding \(y_i\).

The conditions on roots and \(n\) for subsequent algorithms

The rules below concern: when is it possible to proceed with transformation as described above?

  1. \(n\) must have an inverse.
  2. \(w_n^n=1\)
  3. \(w_n^{n/2}=-1\)
  4. There must exist at least \(n\) distinct roots of unity: \(w_n^i \neq w_n^j\) for \((i \neq j) \land (0 \le i,j \lt n)\).

The sequence of roots I: NTT

Last question needed to ultimately solve the problem is: what is \(w_n^1\)? In the second version of the problem, the coefficients have to be calculated \(\pmod m\). If \(m\) has a primitive root \(p\), then:

  1. One can find some primitive root \(p\) rather rapidly.
  2. If \(p\) is a primitive root, then \(p^{\phi(m)/2}=m-1\) - that's because:
    • \(m-1\) is coprime to \(m\) - so, there exists exponent \(z\) such that \(p^z=m-1\).
    • \((m-1)^2=m^2-2m+1 \equiv 1 \equiv p^{\phi(m)} \pmod m\).
    • For any \(z \not\equiv \frac{\phi(m)}{2} \pmod {\phi(m)} \land z \not\equiv 0 \pmod {\phi(m)}\) occurs: \(p^{2z} \not\equiv p^{\phi(m)} \equiv 1 \pmod m\) - so only possible exponent \(z\), such that \((m-1) \equiv p^z \pmod m\) is \(\frac{\phi(m)}{2}\).
  3. One can notice, that if \(\phi(m)=2^kx\), then \(n|2^k\) implies that, \(w_n^1=p^{x2^k/n}=p^{\phi(m)/n}\) can be used as a root - because, as \(p\) is a primitive root, values \(w_n^k\) will be distinct; becuse \(w_n^{n/2} \equiv m-1 \pmod m\), and \(w_n^n \equiv 1 \pmod m\).
  4. Note, that \(m\) does not have to have a primitive root, however, in this case one needs to find such number \(g\), whose \(ord_m(g)=2^kx\), where \(2^k \ge n\) - and such number may not exist, finding it can also become problematic. Furthermore, \(w_n^{n/2}\) does not necessarily need to be equal to \(m-1\), because not for every value \(x\) coprime to \(m\) there will be exponent \(z\) such that \(g^z \equiv x\)
  5. Furthermore, \(n\) must have an inverse \(\pmod m\) - as \(n=2^k\), then \(gcd(2, m)=1\)

The sequence of roots II: DFT

In the first version of the problem, the coefficients are calculated as they are. Instead of finding primitive root, Euler's identity: \(e^{2\pi x}=cos(x)+i*sin(x)\) is used, and calculations are executed using complex numbers.

  1. \(n\) has an inverse \(\frac{1}{n}\) in real numbers (then also in complex numbers).
  2. Let \(w_n^k=e^{2\pi k/n}\) - then \(w_n^n=e^{2 \pi n/n}=e^{2 \pi}=cos(2\pi)+i*sin(2\pi)=1+i*0=1\)
  3. \(w_n^{n/2}=e^{2\pi n/(2n)}=e^{\pi}=cos(\pi)+i*sin(\pi)=-1+i*0=-1\)
  4. The roots \(w_n^k\) are distinct, which is implied by \(\lt sin(x), cos(x) \gt\) being diffrent for all \(x \in \lt 0;2\pi \gt \) - by contradiction:
    • Functions \(sin(x)\) and \(cos(x)\) have different pair of signs in each quarter - so if \(x \neq y \land sin(x)=sin(y) \land cos(x)=cos(y)\), then they must lie in the same quarter.
    • Functions \(sin(x)\) and \(cos(x)\) are monotonically increasing/decreasing in single quarter - so if \(x \neq y \land sin(x)=sin(y) \land cos(x)=cos(y)\), then they must lie in different quarters, which contradicts above.
    • For \(sin(x)\) and \(cos(x)\) in points \(0, \frac{\pi}{2}, \pi, \frac{3\pi}{2}\) one can check property himself.

Problem II - more-or-less standard knapsack dp

Joshua has sequence \(S\) of \(n\) candies. \(i\)-th candy has some number \(s_i\) written on it. For each \(d \in \lt 0;\sum_{i=0}^{n-1} \gt\): in how many ways Joshua can chose some subset of his candies in such way that sum of numbers written on candies in this set is equal to \(d\)? Two ways are different if there is \(k\), such that one set contains \(k\)-th candie while the second don't. Output result \(\pmod {998244353}\) $$ n, \sum_{i=0}^{n-1} s_i \le 10^5 $$

Standard dp solution

  1. The sum of values on candies in a set will be called later set sum
  2. One can notice, that if one can find a number of ways to create subset of set sum with elements of sequence \(S\) up to index \(i\), then one can find number of ways to create subset of any set sum with elements up to index \(i+1\) using fact, that either an element is not chosen (then set sum stays the same) or chosen (then set sum gets larger by \(s_{i+1}\))
  3. Let \(dp[k][x]\) denote number of ways to chose subset with set sum equal to \(x\) using first \(k\) candies. Then \(dp[k][x]=dp[k-1][x]+dp[k-1][x-s_k]\) provided, that \(x-s_k \ge 0\) - subset with set sum \(x\) can be created by not adding anything to existing set or adding \(s_k\) to a set with set sum equal to \(x-s_k\). At the beginning, \(dp[0][0]=1\) - one can create set of value 0 out of empty set in exactly one way.
  4. One can lower the amount of memory needed to solve this task - as only value of \(dp[n][x]\) for all \(x\) are interesting, first dimension is not needed. One can remove it without temporary array by updating values from highest to lowest: \(dp[x]=dp[x]+dp[x-s_k]\) for \(x\) from \(\sum_{i=0}^{k-1} s_i\) to \(s_k\), because one always uses only information present iteration before.
  5. This method works in \(O(n \sum_{i=0}^{n-1} s_i)\), which is unsatisfying - however, it is more flexible than solution that soon will be presented.

Generating functions

One can notice, that it the formula above is in fact polynominal multiplication, where \(dp\) represents a polynominal multiplied by \(x^{s_k}+1\).

Let coefficients \(a_i\) and \(b_i\) of polynominal \(a_mx^m+a_{m-1}x^{m-1}+\dots+a_0\) and \(b_lx^l+b_{l-1}x^{l-1}+\dots+b_0\) represent number of ways to create subsets of set sum equal to \(i\) for two disjoint sets \(A, B\). Then, multiplying those polynominals leads to: $$(\sum_{i=0}^{m} a_ix^i) (\sum_{j=0}^{l} b_jx^j) = \sum_{i=0}^{m} \sum_{j=0}^{l} a_ib_jx^{i+j}$$ Thus:

  1. Value of coefficient by \(x^k\) is equal to \(\sum_{i=0}^{k} a_ib_{k-i}\) assuming, that \(\forall_{v \gt m} a_v=0\) and \(\forall_{w \gt l} b_w=0\). It is equal to number of ways to choose certain amount of elements with given set sum from set \(A, B\) in such way, such that the sum of their set sums is equal to \(k\)
  2. By rule of product, number of ways to choose \(i\) elements from set \(A\) and \(k-i\) elements from set \(B\) is equal to \(a_ib_{k-i}\)
  3. To choose \(k\) elements from both sets one has to choose some elements from set \(A\) and some from set \(B\) (perhaps 0, but not more than \(k\) from any set).
  4. What follows, the coefficient by \(x^k\) is equal to number of ways to choose some elements from set \(A\) and some from set \(B\) in such way, that the sum set of union of those subsets is equal to \(k\).

So, one can represent solution to the stated problem as follows: $$(x^{s_0}+1)(x^{s_1}+1) \dots (x^{s_{n-1}}+1)$$ And coefficient by \(x^d\) in resulting polynominal is answer to the problem

Proper utilization of forementioned observation - replacing dp with NTT

There is still one obstacle one has to remove in order to bathe this problem solutions in glorious capabilities of NTT: how to multiply polynominals, so that complexity will become significantly lower than for dp approach? Multipying them one by one will result in disastrous \(O(n\log(n)(\sum_{i=0}^{n}s_i))\) without better estimation.

Before constructing lemma, let \(m=\sum_{i=0}^{n-1}s_i\).

Let \(pol_{0, i}=x^{s_i}+1\). Then, what if the polynominals are multiplied in layers, in such way, that \(pol_{level,i}=pol_{level-1, 2i}*pol_{level-1, 2i+1}\) (or \(pol_{level,i}=pol_{level-1, 2i}\) for odd number of polynominals in previous layer in the last polynominal of the layer)?

Complexity of such approach

  1. In layer 0, all polynominals have form \(pol_{0,i}=x^{s_i}+1\).
  2. There will be at most \(\lceil log_2(n) \rceil+1\) layers, as in each layer number of polynominals decreases two times (possibly without one element).
  3. In layer \(x\), number of operations can be estimated by
    \((s_0+s_1+\dots+s_{2^x-1})\log(s_0+s_1+\dots+s_{2^x-1})+(s_{2^x}+s_{2^x+1}+\dots+s_{2*2^x-1})\log(s_{2^x}+s_{2^x+1}+\dots+s_{2^x-1})+\dots+(s_{c*2^x}+s_{c*2^x+1}+\dots+s_{n-1})\log(s_{c*2^x}+s_{c*2^x+1}+\dots+s_{n-1}) \le\) \(\le (s_0+s_1+\dots+s_{2^x-1})\log(m)+(s_{2^x}+s_{2^x+1}+\dots+s_{2*2^x-1})\log(m)+\dots+(s_{c*2^x}+s_{c*2^x+1}+\dots+s_{n-1})\log(m)=\) \(= (s_0+s_1+\dots+s_{2^x}+s_{2^x+1}+ \dots + s_{n-1})log_2(m)\) \(= mlog_2(m)\)
  4. Adding this information to estimation of number of layers, one can conclude, that this algorithm works in \(O(m\log_2(m)\log_2(n))\), which is compelling solution for given problem.

Summary

  1. Two polynominals of degrees \(a, b\) with \(a+b=n\) can be multiplied in \(O(nlog(n))\) using DFT or NTT. DFT uses imaginary numbers and may lead to imprecise results because of rounding errors; NTT gives precise results and can be calculated modulo some prime number \(p\), for which \(\exists_k 2^k \gt n \land 2^k | p-1\). In order to apply NTT, one has to find (or know) primitive root \(\pmod p\).
  2. Certain solutions to combinatorical problems can be represented by generating functions - polynominals - and sometimes, by product rule, multiplying those polynominals leads to a polynominal solving greater instance of a problem - this sometimes leads to possibility of solving problem using NTT instead of standard dp approach.
  3. Polynominal multiplication may be treated as answer to the problem: "There are two disjoint sequences of natural numbers \(A\) and \(B\) of size \(n, m\) respectively. Number of ways to select some (possibly 0) elements of sequence \(A\) so that their sum is equal to \(i\) is equal to \(a_i\), analogously \(b_i\) for sequence \(B\). In how many ways one can select some elements of union of those sequences C in order to obtain set with sum of elements \(j\) for every \(j \in \lt 0;n+m \gt\)?" The answer for each \(j\) will be denoted as \(c_j\). Then problem can be represented as \((a_nx^n+a_{n-1}x^{n-1} \dots a_0)(b_mx^m+b_{m-1}x^{m-1}+b_0)=c_{n+m}x^{n+m}+c_{n+m-1}x^{n+m-1}+ \dots +c_0\). Which leads back to multiplying a polynominal.
