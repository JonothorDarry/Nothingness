Methodological Manifesto

What can be said about based website presenting information about processes/ideas possible to model mathematically?

  1. Sense of finiteness - there is limited number of structured domains divided into articles.
  2. Sense of completeness - every argument, that had to be posited was posited.
  3. Homogenity of structure - every single article has same outer structure.
  4. Identity of an article - every article has some elements distinguishing it from other articles.
  5. Dual provability
    • Every proof from a given domain vital to prove shown theorem should be shown explicitly.
    • Amount of theorems without proof outside of given domain left unproved should be as small as possible.
  6. Visual representation - A process or idea, that can be put in visualization shall be visualized if possible.

While this website is not very based, it can become very based sooner or later.

The aim of this website is to make something like https://cp-algorithms.com/ - but containing more thorough prooving, visualizations and well-structured.

Visualizations

Currently, there are 3 types of visualizations:

  1. Showing a process: Four buttons 'begin' (starts a visualization with given input), 'next move', 'previous move', 'finish'.
  2. Showing an idea: One button 'show' showing some offspring of an idea.
  3. Showing a query mechanism: Two visualizations: one shows preprocessing, second shows results of asking query; first has to be finished before second.

Begin gets input and starts process with given input. Finish executes all remaining moves in the algorithm, next/previous executes next or previous move in an algorithm. Note, that only clicking 'Begin' and 'Show' buttons results in using input in any form.
